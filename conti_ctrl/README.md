# E5000 连续插补测试器

这是独立于 `pos_ctrl` 的 Qt Widgets 工程，只覆盖 E5000 连续插补的前两个验证阶段，不包含 CSP。

## 当前测试方式

- 阶段一：选择两个不同的轴；主动轴沿五次多项式位移，保持轴写入其当前绝对位置。两轴仍属于同一个连续插补坐标系，因此能够验证 `open_list / line_unit / start_list`、前瞻、`mark`、缓冲区和停止流程。
- 阶段二：两个所选轴都沿同一个五次多项式比例运动，用于验证双轴同步。
- 上位机轨迹生成器每 `10 ms` 放入一个理论点，作为后续慢周期控制输出的模拟。它不是 EtherCAT 的硬实时周期；控制卡通过自身连续插补缓冲区执行已预压的轨迹。

## 运行顺序

1. 确认急停、限位和机械安全条件已经就绪；根据实机填写轴号、位移、速度等参数。
2. 点击“初始化控制卡”。
3. 选择两个不同轴并点击“使能所选轴”。
4. 先以小位移、低速度运行阶段一，再运行阶段二。
5. 观察 `current / pushed mark`、估算缓冲段数、`remainSpace` 和运行日志。
6. 通过“减速停止”正常停止；仅在紧急情况使用“立即停止”。关闭程序前先停止并关闭控制卡。

## 关键约束

- `dmc_conti_set_lookahead_mode` 在 `dmc_conti_open_list` 之前调用。
- 轨迹使用 `dmc_conti_line_unit` 的绝对模式；启动时读取两个轴的当前位置作为绝对轨迹起点。
- `pushedMark - currentMark` 是段数估算，不代表严格毫秒数。
- 默认参数仅用于软件骨架验证，不能直接视为实机安全参数。

## 构建

工程默认从 `../pos_ctrl/third_party/leadshine` 引用工作区现有的 `LTDMC.h/.lib/.dll`，但不会修改 `pos_ctrl`。在 Qt 6.8.3 MSVC 2022 x64 开发环境中运行：

```bat
qmake conti_ctrl.pro
nmake
```
