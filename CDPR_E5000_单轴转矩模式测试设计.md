# CDPR E5000 单轴转矩模式测试设计

## 1. 目标与边界

本页只用于验证单个 EtherCAT 伺服轴的转矩模式调用、实际转矩反馈和安全停止，不参与速度闭环、连续插补或多轴协同控制。

调用依据：

- 《雷赛 EtherCAT 控制卡转矩功能应用说明》中的 `nmc_torque_move`、`nmc_change_torque`、`nmc_get_torque` 与 `dmc_stop`；
- 原工程 `0523_final` 的转矩换算、220Bh 速度限制写入及独立监测逻辑。

## 2. 分层与线程

- UI 线程：采集参数、显示状态、以 20 Hz 左右批量刷新曲线；
- `ContiWorker` 控制线程：转矩测试状态机、Trace/转矩反馈监测和安全判定；
- `E5000HardwareInterface` 独占硬件线程：唯一允许调用 LTDMC 的边界。

目标转矩不会在每个监测周期重复下发：

- 启动时调用一次 `nmc_torque_move`；
- 用户点击“在线更新转矩”时调用一次 `nmc_change_torque`；
- 监测周期只读取 Trace 与 `nmc_get_torque`。

## 3. 单位换算

CiA402 转矩值按千分额定转矩表示：

\[
TorqueRaw=\operatorname{round}\left(
\frac{Torque_{Nm}}{RatedTorque_{Nm}}\times1000
\right)
\]

实际转矩反算为：

\[
Torque_{Nm}=\frac{TorqueRaw}{1000}\times RatedTorque_{Nm}
\]

原工程额定转矩为 45 N·m，界面保留为可配置参数。位置和速度统一按角度显示；Trace 原始位置仍固定使用 500.622 pulse/deg 换算。

## 4. 启动流程

1. 初始化控制卡并确认测试轴在线；
2. 使能测试轴；
3. 根据实际硬件需要，手动写入并读回转矩模式速度限制 OD；
4. 设置非零目标转矩，确认命令限幅、最大行程、最大速度和最长运行时间；
5. 点击“开始转矩测试”；
6. 程序重配该轴 Trace，取得有效首帧；
7. 记录实际起点，根据转矩方向计算绝对位置限位；
8. 调用 `nmc_torque_move` 进入转矩模式；
9. 周期监测实际转矩、位置、速度、轴状态和 Trace 新鲜度；
10. 停止时调用 `dmc_stop`，由控制卡使伺服返回位置模式。

程序不会自动使能轴，也不会在启动转矩测试时自动改写伺服 OD。

## 5. 速度限制 OD

不同驱动器的实现并不统一，因此界面显式区分：

- 220Bh：原工程雷赛驱动器用法，界面输入 rpm，程序按当前编码器 180224 pulse/rev 换算为 pulse/s；
- 6080h：转矩功能说明中的 CiA402 速度限制，界面直接输入驱动器原生整数值。

两者都只在“写入并读回”按钮触发时操作。6080h 的实际单位必须以当前伺服驱动器手册为准。

## 6. 安全规则

以下任一条件触发停止：

- 目标转矩超过命令限幅；
- `nmc_get_torque` 换算后的实际转矩超过反馈保护上限；
- 实际相对位移达到最大行程；
- Trace 实际速度超过软件速度上限；
- Trace 在设置时间内没有新帧；
- 轴状态机不是 4 或轴错误码非零；
- 达到最长运行时间；
- 读取实际转矩或控制卡 API 调用失败。

硬件位置限位启用时，`nmc_torque_move` 同时接收按运动方向计算的绝对限位；软件相对行程监测始终保留。运行中禁止直接改变转矩符号，因为硬件限位方向在启动时已经固定；反向前必须先停止并重新启动。

## 7. 首轮测试建议

- 脱离高负载机构或采用可靠机械限位；
- 目标转矩从额定转矩的很小比例开始；
- 命令限幅、行程、速度和最长运行时间取保守值；
- 先验证“立即停止”和“减速停止”；
- 再验证 `nmc_get_torque` 的正负号、比例和重复性；
- 最后测试同方向在线调整转矩。
